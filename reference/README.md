# `reference/` — test corpus & fixtures

This directory holds the validation corpus. Most of it is **regenerable or
local-only** and is *stripped* when a lean public source repo is seeded
(see [`PUBLIC-SEED-MANIFEST.md`](./PUBLIC-SEED-MANIFEST.md)). Only the small
DXT1 byte-gate oracles are irreplaceable and ship verbatim.

The methodology behind the corpus is in
[`CORPUS-METHODOLOGY.md`](./CORPUS-METHODOLOGY.md).

## What ships in the public seed (~5 MB + small DXT1 source)

| Path | Why it must ship |
|---|---|
| `dxt1/ffmpeg.mov` | DXT1 byte-gate oracle. **Byte-identity-locked** to FFmpeg 8.1.1 / libavcodec 62.28.101 (`-c:v dxv -format dxt1`); a different FFmpeg build will not reproduce its bytes. |
| `dxt1/alley.mov` | DXT1 byte-gate oracle. **Irreproducible by script** — a Resolume Alley export (proprietary GUI, no CLI/headless path). |
| `dxt1/source/frame_0001..0030.png` | The byte gate feeds these through `DXT1Encoder` → `DXVMOVWriter` and compares the result against the two oracles above. |
| `*/FINDINGS.md`, `*/PHASE-*-RESULTS.md`, `CORPUS-METHODOLOGY.md`, `endpoint-search-study/FINDINGS.md` | Byte-archaeology documentation. |

## What is regenerable via `scripts/make-corpus.sh`

| Group | Tool needed | Notes |
|---|---|---|
| `synthetic-corpus/source/*.png` | none | Deterministic CoreGraphics draw — byte-for-byte reproducible. |
| `dxt1/source/source.mov` + `frame_*.png` | **FFmpeg** | `testsrc2` filter, 1920×1080@30, 30 frames. Inputs only — *not* version-byte-sensitive. |
| `dxt5/ycg6/yg10/source/*` | **FFmpeg** | `testsrc2` (+ synthesized alpha mask for dxt5/yg10); per-variant commands in each `FINDINGS.md`. |

## What requires Resolume (unscriptable)

- `dxt1/alley.mov`, and the archival `*/alley.mov` for the other variants — all
  Resolume Alley exports. There is no automated path; these ship (dxt1) or are
  simply absent (others), and dependent tests **skip** when absent.

## What requires local-only media (not in any repo)

- `realworld-corpus/`, `realworld-yg10-corpus/`, `realworld-dxt5-paired-corpus/`,
  `dxt5/realworld-source/`, `dxdi/sample.mov`, `hap-source/`, `hap-audio/`,
  `hapm/`, `fps/*.mp4` — decoded/derived from clips on the author's machine.
  `scripts/make-corpus.sh [path]` accepts a source clip for the basic real-world
  generator; otherwise these are skipped.

## GlEnc-produced artifacts (regenerated by the encoder tests themselves)

- `*/glenc.mov` — GlEnc's own output, committed as SSIM/comparison artifacts.
  Re-saved by the `…AndSaveReference` encoder tests. Stripped from the seed;
  tests that read them **skip** when absent.

## Fixture-absence behavior

Every test that consumes a non-shipping fixture **skips cleanly** (XCTSkip) with
a message naming the missing file and how to regenerate it. A fixture-less
checkout runs the four self-contained byte-gate members plus the two DXT1 oracle
tests (oracles ship) green, and skips the rest — it never *fails* for a missing
fixture.
